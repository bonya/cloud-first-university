AWS Regions and Availability Zones

  • AWS is hosted in multiple locations worldwide - Regions
  • Region consists of multiple isolated locations - Availability Zones
  • You can choose Region and Availability Zone

Amazon Machine Image (AMI)

  • AMI is a templates that contains configuration - operation system and software packages
  • AMI is used to launch an instance - virtual server on AWS
  • There are AMIs provided by amazon and AMIs created by others
  • You can create your own AMI to launch instances
  • Amazon offer a marketplace for AMIs

Instance Type

  • Instance type is combination of CPU, Memory ans Storage
  • By choosing right instance type, you get an optimal mix of resources for you application by minimizing costs
  • T2, M3 - general purpose
  • C3, C4 - compute optimized
  • R3 - memory optimized
  • G2 - GPU for CUDA
  • I2 - storage optimized high I/O
  • D2 - dense storage optimized

Storage Configuration

  • Add additional volumes

Instance Tags

  • Provide human-readable instance Name
  • Add extra tags to store metadata

Security Groups

  • Security Group (SG) is a virtual firewall
  • SG can be used by multiple instances
  • SG is created in particular Region
  • Changes in SG are applied to all associated instances

Instance Lifecyle

Instance pricing models

  • On Demand Instace
  • Reserved Instance
  • Spot Price

On Demand Instance

  • Pay for capacity by hours used
  • No long-term commitment and upfront fees
  • Used to handle spikes in workload
  • Development or Staging

Reserved Instance

  • Upfront fees with mid- and long-term contracts
  • Up to 50% cheaper than on demand instances
  • Used to handle stable workloads

Spot Instance

  • (Normally) 50-90% cheaper than on demand
  • Can be terminated if somebody bids higher price
  • Applications feasible only at very low computing prices
  • Applications can stand unexpected server termination

EBS: Elastic Block Store

  • EBS is a persistent storage for you data and applications
  • EBS is attached as a volume to your instance
  • You can create Snapshots from EBS volumes
  • New EBS volume can be created from Snapshots
  • Snapshots are incremental
  • Provisioned IOPS offer consistent performance for high workloads
  • EBS offers build-in Encryption for selected isntance types

Summary

  • Instance is a virtual machine running in cloud
  • Instance is started in particular Region and Availability Zone
  • AMI is a template for creating new instances
  • Instance type is a combination CPU, RAM and Storage
  • Instance can be added to VPC, IAM Role etc.
  • Use EBS volumes to store you data
  • Use Security Group open instance ports
  • Tags name your instance and provide meta data
